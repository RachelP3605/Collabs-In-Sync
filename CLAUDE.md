# Collab Directory LP — CLAUDE.md

## Project Context

Landing page for Collabs in Sync (collabsinsync.com). Goal: Convert cold visitors from Reddit/Instagram into early-access signups.

**Buyer:** Skilled songwriter, producer, or singer who tried paying for collaboration, got priced out, now hunting on Reddit/DMs. Frustrated. Anxious. Stuck.

**Status:** Cold launch, pre-launch (building early-access list). Phase 3 in progress.

---

## Locked Fundamentals

**The One Promise:** "The Platform Built for Profit-Sharing Music Creators"

**The Mechanism:** "No hunting on Reddit. No expensive hiring. Just match with the collaborators you need."

**The Rule of One:**
- Idea: You don't have to hunt or hire. You match.
- Feeling: Relief. Finally, a place where everyone wants exactly what you're offering.
- Action: Get early access.

## RULES FOR THIS BUILD

### Before You Build
- **Ask clarifying questions first.** Do not guess or assume.
- **Read the strategy docs before editing site.** In order: SURFACE-RESEARCH.md → LANDING-PAGE-COPY.md (if exists) → DESIGN.md → SURFACE-BRIEF.md → then edit site/.

### What Goes Where
- **All work stays inside this project folder.** No files scattered in other places.
- **Copy goes in LANDING-PAGE-COPY.md.** Not drafted in the site files.
- **Design decisions go in DESIGN.md.** Record choices as you make them.
- **Feedback goes in REVIEW-NOTES.md.** Track revisions by date.
- **Deploy/build notes go in README.md.** How to run it, where it lives, status.

### Before Any UI Decision
- **Run frontend-design skill.** Do not make layout, color, or component choices without it.

### Never Invent
These are absolutes. Do not create, guess, or fill in gaps:
- Proof (numbers, percentages, statistics)
- Testimonials or customer quotes
- Logos, screenshots, or images (use mockups only)
- Guarantees or promises
- Customer success stories

Use only what you actually have. If it's missing, ask Rachel before proceeding.

### Guard the Focus
- **One buyer.** Not multiple segments.
- **One promise.** Not a feature list.
- **One action.** Not multiple CTAs competing.

Protect this ruthlessly. If anything tries to creep in, cut it.

### Verification
- **Check mobile before calling done.** Responsive is non-negotiable.
- **Test the golden path.** Does the CTA work? Does early access signup flow?

### Communication
- **Ask clarifying questions before building.** Do not guess.
- **Show work before finalizing.** Get sign-off at each phase gate (CLAUDE.md gate file).
- **Report blockers immediately.** If missing info, say so upfront, don't work around it.
